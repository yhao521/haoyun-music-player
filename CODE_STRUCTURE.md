# 项目代码结构说明

## 📁 文件组织结构

为了提高代码的可维护性和可读性，项目采用了模块化设计，将原本庞大的 `main.go` 拆分为多个功能模块。

### 根目录文件（package main）

所有以下文件都使用 `package main`，因为它们需要访问 Wails 应用的全局状态：

#### 1. **main.go** - 应用入口
- **职责**：应用启动流程编排
- **主要功能**：
  - 调用各模块初始化函数
  - 协调应用启动顺序
  - 优雅关闭处理
- **行数**：~80 行（精简后）

#### 2. **app_init.go** - 应用初始化
- **职责**：核心组件初始化和配置加载
- **主要功能**：
  - 事件注册（`registerEvents`）
  - 配置管理器初始化
  - 国际化模块初始化
  - 音乐服务和依赖管理器创建
  - Wails 应用实例创建
  - 依赖工具异步检查
- **关键函数**：
  - `initializeApp()` - 初始化所有核心组件
  - `checkDependencies()` - 后台检查依赖工具状态

#### 3. **window_manager.go** - 窗口管理
- **职责**：所有窗口的创建、显示/隐藏控制
- **主要功能**：
  - 创建主窗口、浏览窗口、喜爱音乐窗口、设置窗口
  - 窗口关闭拦截（统一隐藏行为）
  - 窗口可见性切换
  - 测试消息发送
- **关键函数**：
  - `createAllWindows()` - 创建所有窗口
  - `toggleWindowVisibility()` - 切换窗口显示/隐藏
  - `registerWindowCloseHooks()` - 注册关闭拦截器

#### 4. **tray_menu.go** - 托盘菜单管理
- **职责**：系统托盘图标和菜单的完整管理
- **主要功能**：
  - 托盘图标初始化
  - 所有菜单项创建（播放控制、窗口管理、音乐库、工具等）
  - 动态菜单构建（音乐库列表、依赖工具状态）
  - 菜单重建（语言切换时）
  - 正在播放信息显示更新
- **关键函数**：
  - `createTrayAndMenu()` - 创建托盘和菜单
  - `buildMusicLibMenu()` - 动态构建音乐库菜单
  - `buildToolsMenu()` - 动态构建工具菜单
  - `rebuildTrayMenu()` - 重建整个菜单（用于语言切换）
  - `updateNowPlayingItem()` - 更新正在播放显示

#### 5. **event_handlers.go** - 事件处理器
- **职责**：所有 Wails 事件的监听和处理
- **主要功能**：
  - 播放控制事件（播放/暂停、上一曲、下一曲、播放模式）
  - 窗口管理事件（打开/关闭窗口）
  - 设置相关事件（语言切换、配置更新、获取配置、重启应用）
  - 音乐库事件（压缩音乐库）
  - 依赖管理器回调设置
- **关键函数**：
  - `registerEventHandlers()` - 注册所有事件处理器
  - `setupDependencyManagerCallback()` - 设置依赖管理器回调

#### 6. **main_menu.go** - 主菜单栏
- **职责**：应用程序顶部菜单栏的创建
- **主要功能**：
  - 创建 File、Music、Edit、View、Help 等菜单
  - 设置菜单项快捷键
  - 开发菜单（重载、开发者工具）
  - 打开应用目录功能
- **关键函数**：
  - `createMenu()` - 创建完整的主菜单
  - `OpenDir()` - 打开应用运行目录

### backend 目录

保持原有的后端业务逻辑不变：

```
backend/
├── pkg/
│   ├── config/      # 配置管理
│   ├── i18n/        # 国际化
│   ├── file/        # 文件操作
│   └── utils/       # 工具函数
├── *.go             # 各种业务逻辑服务
```

## 🎯 重构优势

### 1. **可维护性提升**
- 每个文件职责单一，易于理解和修改
- 新功能可以添加到对应的模块文件中
- 减少代码冲突（多人协作时）

### 2. **可读性提升**
- `main.go` 从 1800+ 行精简到 ~80 行
- 清晰的函数命名和注释
- 快速定位特定功能的代码位置

### 3. **可扩展性提升**
- 新增窗口：在 `window_manager.go` 中添加
- 新增事件：在 `event_handlers.go` 中注册
- 新增菜单项：在 `tray_menu.go` 或 `main_menu.go` 中添加

### 4. **调试便利性**
- 问题可以快速定位到具体模块
- 日志输出更清晰（带模块标识）
- 单元测试更容易编写

## 📝 开发指南

### 添加新窗口
1. 在 `window_manager.go` 中：
   - 声明窗口变量
   - 创建 `createXXXWindow()` 函数
   - 在 `createAllWindows()` 中调用
   - 在 `hasOtherVisibleWindows()` 中添加判断
   - 在 `registerWindowCloseHooks()` 中注册关闭拦截

### 添加新事件
1. 在 `app_init.go` 的 `registerEvents()` 中注册事件
2. 在 `event_handlers.go` 中添加处理函数
3. 在 `registerEventHandlers()` 中调用

### 添加新菜单项
**托盘菜单**：
1. 在 `tray_menu.go` 中声明菜单项变量
2. 在 `createTrayMenuItems()` 中创建菜单项
3. 在 `buildInitialTrayMenu()` 中组装到菜单
4. 在 `rebuildTrayMenu()` 中更新标签（支持多语言）

**主菜单**：
1. 在 `main_menu.go` 的 `createMenu()` 中添加

### 修改窗口行为
- 所有窗口相关逻辑都在 `window_manager.go` 中
- 包括显示/隐藏、关闭拦截、焦点管理等

## ⚠️ 注意事项

1. **包名统一**：所有拆分文件必须使用 `package main`
2. **全局变量**：共享的全局变量在 `app_init.go` 中声明
3. **循环依赖**：避免模块间的循环调用
4. **闭包引用**：注意闭包中引用的变量作用域
5. **Wails 要求**：`main()` 函数必须在项目根目录

## 🔄 后续优化建议

1. **进一步拆分**：如果某个文件超过 500 行，可以考虑继续拆分
2. **接口抽象**：为大型模块定义接口，便于测试
3. **配置化**：将硬编码的菜单项、窗口配置提取到配置文件
4. **单元测试**：为独立函数编写单元测试
