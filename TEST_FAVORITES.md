# 喜爱音乐功能测试指南（独立窗口版）

## 前置条件
1. 确保已安装依赖：
   ```bash
   cd frontend && npm install
   ```

2. 确保已有播放历史记录（至少播放过几首歌曲）

## 测试步骤

### 1. 启动应用
```bash
# 在项目根目录执行
./start.sh  # macOS/Linux
# 或
start.bat   # Windows
```

### 2. 测试托盘菜单访问
1. 查看系统托盘区域，找到 Haoyun Music Player 图标
2. 点击托盘图标打开菜单
3. 查找 "❤️ 喜爱音乐" 菜单项
4. 验证快捷键提示显示为 `Cmd+H` (macOS) 或 `Ctrl+H` (Windows/Linux)
5. 点击该菜单项

**预期结果**:
- ✅ **新的独立窗口**自动显示并获得焦点
- ✅ 窗口标题为 "喜爱音乐 - Haoyun Music Player"
- ✅ 窗口大小为 900x700
- ✅ 页面直接显示喜爱音乐列表（URL: `#/favorites`）
- ✅ 主窗口保持原有状态（不受影响）

### 3. 测试快捷键
1. 确保任意窗口处于激活状态
2. 按下 `Cmd+H` (macOS) 或 `Ctrl+H` (其他平台)

**预期结果**:
- ✅ 如果喜爱音乐窗口未显示 → 显示窗口
- ✅ 如果喜爱音乐窗口已显示 → 隐藏窗口
- ✅ 切换流畅，无卡顿

### 4. 测试窗口显示/隐藏切换
1. 第一次点击托盘菜单 "❤️ 喜爱音乐"
   - ✅ 窗口显示
2. 第二次点击同一菜单项
   - ✅ 窗口隐藏
3. 第三次点击
   - ✅ 窗口再次显示

**验证点**:
- ✅ 每次显示时窗口内容正确加载
- ✅ 隐藏后窗口不销毁，只是不可见
- ✅ 可以快速切换，无内存泄漏

### 5. 测试窗口关闭拦截

**预期结果**:

## 已知问题排查

### 问题 1: 点击托盘菜单无反应
**可能原因**: 
- favoritesWindow 未正确初始化
- 窗口变量为 nil

**排查步骤**:
1. 查看终端日志，搜索 "打开喜爱音乐窗口"
2. 检查是否有 "favoritesWindow 为 nil" 错误
3. 确认窗口创建代码已执行（搜索 "✓ Favorites window created"）

### 问题 2: 窗口显示但内容为空白
**可能原因**:
- 路由配置错误
- FavoritesView 组件未正确加载

**排查步骤**:
1. 检查窗口 URL 是否为 `#/favorites`
2. 查看浏览器开发者工具 Console 是否有错误
3. 确认 App.vue 中的路由判断逻辑正确

### 问题 3: 列表为空但应该有数据
**可能原因**:
- 播放历史被清空
- PlayCount 字段为 0

**排查步骤**:
1. 播放几首歌曲
2. 检查 `~/.haoyun-music/history.json` 文件
3. 确认 `play_count` 字段大于 0
4. 点击刷新按钮重新加载

### 问题 4: 关闭窗口后程序退出
**可能原因**:
- 关闭拦截钩子未正确注册

**排查步骤**:
1. 检查终端是否有 "喜爱音乐窗口关闭事件被拦截" 日志
2. 确认 `RegisterHook` 调用在窗口创建后立即执行
3. 检查 `e.Cancel()` 是否被调用

### 问题 5: 样式显示异常
**可能原因**:
- CSS 未正确加载
- 浏览器兼容性问题

**排查步骤**:
1. 清除浏览器缓存
2. 重启应用
3. 检查开发者工具 Console 是否有 CSS 错误

## 性能测试

### 窗口打开时间
- **期望**: < 500ms（从点击菜单到窗口完全显示）
- **测量方法**: 使用计时器或主观感受

### 数据加载时间
- **期望**: < 500ms（100 条记录）
- **测量方法**: 从窗口显示到列表渲染完成的时间

### 内存占用
- **期望**: 额外 < 100MB（相比单窗口版本）
- **测量方法**: 使用系统活动监视器对比打开前后的内存使用

- **期望**: 60 FPS
- **测量方法**: 快速滚动列表，观察是否卡顿

### 多窗口切换
- **期望**: < 200ms（显示/隐藏切换）
- **测量方法**: 快速点击托盘菜单，观察响应速度

## 与单视图版本的对比测试

如果你之前测试过单视图版本，可以对比以下方面：

| 测试项 | 单视图版本 | 独立窗口版本 | 优势 |
|--------|-----------|------------|------|
| 打开速度 | 快（视图切换） | 略慢（窗口创建） | - |
| 用户体验 | 需返回导航 | 直接关闭窗口 | ✅ 独立窗口 |
| 多任务 | 不支持 | 支持多窗口并行 | ✅ 独立窗口 |
| 内存占用 | 低 | 略高 | - |
| 操作便捷性 | 中等 | 高（快捷键） | ✅ 独立窗口 |
| 状态管理 | 简单 | 复杂（需同步） | - |

**结论**: 独立窗口版本在用户体验和多任务处理上更优，适合专业音乐播放器场景。
