# 喜爱音乐功能测试指南

## 前置条件
1. 确保已安装依赖：
   ```bash
   cd frontend && npm install
   ```

2. 确保已有播放历史记录（至少播放过几首歌曲）

## 测试步骤

### 1. 启动应用
```bash
# 在项目根目录执行
./start.sh  # macOS/Linux
# 或
start.bat   # Windows
```

### 2. 测试托盘菜单访问
1. 查看系统托盘区域，找到 Haoyun Music Player 图标
2. 点击托盘图标打开菜单
3. 查找 "❤️ 喜爱音乐" 菜单项
4. 验证快捷键提示显示为 `Cmd+H` (macOS) 或 `Ctrl+H` (Windows/Linux)
5. 点击该菜单项

**预期结果**:
- ✅ 主窗口自动显示并获得焦点
- ✅ 页面导航到喜爱音乐界面
- ✅ URL hash 变为 `#/favorites`

### 3. 测试快捷键
1. 确保主窗口处于激活状态
2. 按下 `Cmd+H` (macOS) 或 `Ctrl+H` (其他平台)

**预期结果**:
- ✅ 立即跳转到喜爱音乐界面

### 4. 测试数据加载
1. 进入喜爱音乐界面后观察
2. 检查是否显示加载动画（如果数据较多）
3. 等待数据加载完成

**预期结果**:
- ✅ 显示加载 spinner（短暂）
- ✅ 成功加载后显示歌曲列表表格
- ✅ 如果没有喜爱音乐，显示空状态提示："暂无喜爱音乐"

### 5. 测试列表展示
检查表格是否正确显示以下列：
- ✅ 排名序号 (#)
- ✅ 歌曲标题
- ✅ 艺术家
- ✅ 专辑
- ✅ 播放次数（金色徽章样式）
- ✅ 时长（mm:ss 格式）
- ✅ 文件大小（KB/MB/GB）

**验证点**:
- 播放次数按降序排列（最多的在最上面）
- 播放次数徽章为金色背景
- 长文本正确截断并显示省略号
- 悬停行有视觉反馈（背景变亮、轻微放大）

### 6. 测试点击播放
1. 点击列表中的任意一首歌曲

**预期结果**:
- ✅ 清空当前播放列表
- ✅ 添加该歌曲到播放列表
- ✅ 开始播放该歌曲
- ✅ 控制台输出 "开始播放: [歌曲名]"

### 7. 测试刷新功能
1. 点击右上角的 "🔄 刷新" 按钮

**预期结果**:
- ✅ 重新加载数据
- ✅ 如果有新播放的歌曲，列表会更新
- ✅ 按钮在加载期间禁用

### 8. 测试返回导航
1. 点击左上角的 "← 返回" 按钮

**预期结果**:
- ✅ 返回主界面
- ✅ URL hash 变为 `#/` 或 `#/main`

### 9. 测试响应式布局
1. 调整窗口大小（缩小宽度）
2. 观察表格行为

**预期结果**:
- ✅ 表格支持横向滚动
- ✅ 滚动条样式与主题协调
- ✅ 不会出现布局错乱

### 10. 测试边界情况

#### 空状态
删除所有播放历史后：
```bash
# 通过托盘菜单 → 清空播放历史
```
然后访问喜爱音乐界面

**预期结果**:
- ✅ 显示空状态图标 🎵
- ✅ 显示提示文字："多听几首歌，它们就会出现在这里哦~"

#### 大量数据
播放超过 100 首不同的歌曲后：

**预期结果**:

- ✅ 底部显示统计信息："共 100 首喜爱音乐"
- ✅ 性能良好，无明显卡顿

#### 错误处理
模拟网络错误或 API 失败（可选）：

**预期结果**:
- ✅ 显示红色错误提示框

## 已知问题排查

### 问题 1: 点击托盘菜单无反应
**可能原因**: 
- mainWindow 未正确初始化
- 事件发射失败

**排查步骤**:
1. 查看终端日志，搜索 "打开喜爱音乐界面"
2. 检查是否有 "已发送导航事件" 日志
3. 确认 mainWindow 不为 nil

### 问题 2: 列表为空但应该有数据
**可能原因**:
- 播放历史被清空
- PlayCount 字段为 0

**排查步骤**:
1. 播放几首歌曲
2. 检查 `~/.haoyun-music/history.json` 文件
3. 确认 `play_count` 字段大于 0
4. 刷新喜爱音乐列表

### 问题 3: 样式显示异常
**可能原因**:
- CSS 未正确加载
- 浏览器兼容性问题

**排查步骤**:
1. 清除浏览器缓存
2. 重启应用
3. 检查开发者工具 Console 是否有 CSS 错误

## 性能测试

### 加载时间
- **期望**: < 500ms（100 条记录）
- **测量方法**: 从点击菜单到列表完全渲染的时间

### 内存占用
- **期望**: < 50MB 额外内存
- **测量方法**: 使用系统活动监视器对比打开前后的内存使用

### 滚动流畅度
- **期望**: 60 FPS
- **测量方法**: 快速滚动列表，观察是否卡顿
